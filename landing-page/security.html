1. Our Security Philosophy

HabitAI is built on a simple principle: your data belongs to you. We designed the entire system around minimizing trust requirements. You should not have to trust us to keep your data safe — because by default, we never have access to it.

Our security model follows a zero-knowledge approach wherever possible. We encrypt data that leaves your device, minimize data collection, and never expose your personal information to third parties.

2. Local-First Architecture

The foundation of HabitAI's security is its local-first architecture. Unlike most habit trackers, your data lives on your device, not on our servers.

  • No account required. The app works fully offline without registration.
  • No data transmission by default. Core app features never make network requests.
  • On-device processing. All habit logic, streak calculations, and analytics run locally.
  • No tracking SDKs. We do not embed any third-party analytics, advertising, or tracking libraries.

3. Encryption Standards

HabitAI uses industry-standard encryption to protect your data at every layer:

Layer Standard Details
Local storage AES-256 All on-device habit data is encrypted using AES-256 via the device's secure enclave or equivalent hardware-backed keystore.
Data in transit TLS 1.3 All network communication uses TLS 1.3 with strong cipher suites and certificate pinning.
Cloud sync AES-256 + E2EE Synced data is encrypted at rest with AES-256 and additionally with end-to-end encryption keys derived from your authentication token.
AI processing In-memory only Voice transcripts are processed in memory and not persisted to disk on our servers beyond the duration of the request.

End-to-End Encryption (E2EE)

When cloud sync is enabled, your habit data is encrypted end-to-end. This means that even we — the developers of HabitAI — cannot read your habit content on our servers. Encryption keys are generated on your device and are never transmitted or stored by us in plaintext.

4. Cloud Sync Security

Cloud sync is an optional opt-in feature. When you enable it, we implement the following protections:

  • Authentication: Sign-in uses Google Sign-In with OAuth 2.0. We never see or store your Google password.
  • Authorization: Access tokens are short-lived and refreshed securely. We validate all API requests against your authenticated identity.
  • Data isolation: Each user's data is stored in a logically isolated partition. No cross-account access is possible.
  • Access revocation: You can revoke sync access at any time from within the app or through your Google Account settings.

5. Network Security

When the app makes network requests (only for optional features like cloud sync and AI processing):

  • Certificate pinning: The app verifies server certificates against pinned fingerprints to prevent man-in-the-middle attacks.
  • API authentication: All API endpoints require authentication via signed JSON Web Tokens (JWT) with automatic expiry.
  • Rate limiting: API endpoints are rate-limited to prevent abuse and brute-force attacks.
  • Minimal attack surface: Our API has only the necessary endpoints. No unnecessary data is exposed.

6. No Tracking, No Analytics

HabitAI does not include:

  • Third-party analytics frameworks (Google Analytics, Firebase, Mixpanel, etc.)
  • Advertising SDKs or ad trackers
  • Data broker integrations
  • Fingerprinting or device profiling
  • Behavioral tracking or session recording

The only anonymous data collected (with your permission) is opt-in crash reporting using a privacy-preserving service that does not associate crash data with any user identifier.

7. Code Integrity & Updates

  • App signing: All HabitAI releases are code-signed and verified by the App Store and Google Play.
  • Dependency auditing: We regularly audit our dependencies for known vulnerabilities using automated tooling.
  • Secure development: We follow secure coding practices, including input validation, parameterized queries (where applicable), and regular code review.

8. Vulnerability Reporting

We take security seriously. If you discover a security vulnerability in HabitAI, please disclose it responsibly:

Email: security@habitai.in
Expected response time: Within 72 hours
Policy: We will investigate all reports promptly and will acknowledge your contribution with your consent. We ask that you do not publicly disclose vulnerabilities until we have had a reasonable opportunity to address them.

We commit to:

  • Acknowledging receipt of your report within 72 hours.
  • Providing a timeline for resolution.
  • Keeping you informed as we address the issue.
  • Recognizing your contribution (with your permission).
